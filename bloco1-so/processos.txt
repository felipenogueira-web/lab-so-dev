# PARTE C — Questões de síntese

# 1. O campo PPID em /proc/$$/status indica o processo pai do shell.
#    Execute ps -o pid,ppid,cmd e localize o par shell -> processo pai.
#    O que esse encadeamento sugere sobre como os processos sao
#    criados no Linux?

Resposta 1: O encadeamento sugere que processos no Linux não surgem do nada; eles formam uma árvore genealógica. Todo processo (exceto o init/systemd) é criado por um processo pai através de uma chamada de sistema (como fork). Isso demonstra que o sistema operacional organiza a execução de forma hierárquica, onde o pai é responsável por monitorar o ciclo de vida do filho.


# 2. O arquivo /proc/<PID>/status mostra o campo State com uma letra
#    (S, R, Z...). Depois de executar kill $FILHO e rodar ps novamente,
#    o que aconteceu com o processo filho? O que isso te faz pensar  
#    sobre a vida de um processo?

Resposta 2: Após o comando kill, o processo filho é encerrado. Se o pai não estivesse atento, o processo poderia se tornar um "Zombie" (Z). Isso nos mostra que a vida de um processo é um ciclo de estados gerenciados pelo Kernel: ele nasce (fork), executa (R), pode esperar por recursos (S) e, ao "morrer", precisa que seu pai reconheça seu fim para que seus recursos sejam totalmente liberados da tabela de processos.
